# Bash curl parallel requests

Баш скрипт параллельных запросов с помощью curl нужен для того, чтобы протестировать, как веб сервер обрабатывает различное количество параллельных запросов.

## Примерные необходимые системные требования:

1 гб оперативки </br>
2 ядра процессора </br>
10гб жесткий диск </br>

Технологии, которые необходимы для запуска баш скрипта параллельных запросов и работы с ним:
1. linux с bash
2. bc
3. curl

---

## Инструкция по запуску

Эта инструкция поможет вам запустить баш скрипт параллельных запросов с помощью curl:

1. Зайдите в вашу операционную систему linux с bash
2. Перейдите в директирию где вы хотите расположить скрипт, например:
     ```bash
    cd /home/{USER_NAME}/bash-curl-parallel-requests
     ```
3. Если хотите скачать этот скрипт из репозитория через гит
    ```bash
    git clone ...
    ```
   то [Установите git](https://git-scm.com/) (если еще не установлен) </br>
   Либо</br>
   Вы можете скачать этот скрипт из репозитория вручную без гита
4. Проверьте установлен ли у вас curl
    ```bash
    curl --version
    ```
    Если установлен, то пропустите этот шаг
    Если не установлен, то установите
    ```bash
    sudo apt install curl
    ```
    есть и [другие способы установки](https://curl.se/download.html)
5. В скрипте curl-parallel-requests.sh </br>
   - замените значение переменной url на значение вашего урла </br>
   - замените значение переменной totalRequests на значение количества параллельных запросов, которое вы хотите отправить
6. Проверяем что у скрипта curl-parallel-requests.sh есть права на исполнение, если нет, выдаём ему эти права
    ```bash
    chmod +x curl-parallel-requests.sh
    ```
7. Запустите скрипт в терминале командой:
    ```bash
    ./curl-parallel-requests.sh
    ```
8. После получения ответа от всех запросов, в терминале отобразится обработка каждой строки ответа, для составления статистики среднего времени выполнения запросов, примерно так:

    - verboses.log, где будут лежать логи выполнения всех запросов
